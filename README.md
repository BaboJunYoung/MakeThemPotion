# Potion Workshop

작은 마법 포션 가게를 운영하는 캐주얼 주문 제작 게임.

손님이 말풍선으로 원하는 효과를 말하면, 플레이어는 주문 의도를 해석해 재료를 가마솥에 넣고 포션을 완성한 뒤 제출한다. 전체 톤은 `좋은 피자, 위대한 피자`처럼 단순하고 직관적이며, 주문 대사를 읽고 맞는 제작법을 떠올리는 재미에 집중한다.

## 핵심 방향

- 복잡한 성장 시스템보다 `주문 이해 -> 제작 -> 제출 -> 반응` 루프를 우선한다.
- 재료 10개, 포션 15개, 하루 손님 5명 내외로 구성한다.
- 모든 포션은 재료 2~3개 조합으로 만든다.
- 제한 시간, 인벤토리 소모, 업그레이드, 희귀 재료는 MVP 이후로 미룬다.
- 플레이어가 실수해도 게임이 막히지 않게 정답/오답 보상만 다르게 준다.

## 게임 한 줄 설명

마법 가게의 주인이 되어 손님의 애매한 주문을 해석하고, 열 가지 재료를 조합해 열다섯 가지 포션을 만들어 판매하는 포션 제작 시뮬레이션.

## 경영 모드 (발표용 차별점)

단순 주문 게임이 아니라, **제한 시간(120초) 안에 월세 목표(300G)를 모아야 하는 리스크 경영 게임**이다. 핵심 4대 시스템:

- **월세 목표 시스템** — 시간이 끝났을 때 돈이 목표 이상이면 클리어, 미달이면 "가게가 망했다" 엔딩. 정상 재료만 팔아서는 목표에 닿지 못하도록 설계.
- **불법 재료 리스크 시스템** — 재료를 정상/수상(불법)으로 나눈다. 수상한 재료는 수입이 크지만 검사 위험도가 오른다(버튼에 빨간 "수상" 표식). **주문을 틀리면 돈을 빼앗기고, 자금이 음수가 되면 즉시 한울이에게 적발**된다.
- **한울 검사 게이지** — 불법 재료·오답으로 게이지가 오르고, 시간이 지나면 결제 내역 추적으로도 서서히 샌다. 게이지가 높아지면 한울이(=customer3)가 검사하러 등장한다.
- **퇴치 포션 시스템** — 한울이가 등장하면 상단 팝업에 **의심 → 검증(총 15초)** 단계와 남은 시간이 표시된다. 시간 안에 퇴치 포션을 제작하면 잠시 쫓아내지만, 대처가 느리면 적발되어 customer3가 `한울이한테적발` 모습으로 변하며 게임오버. 후반에는 내성이 생겨 더 강한 레시피가 필요하다.

각 시스템은 `game_logic.py`에 동일한 주석으로 표시해 두었다. 효과음(message/correct/wrong/select/click/appearance/inform/caught)으로 피드백을 준다.

> 개발용: 인트로는 **Ctrl + Enter** 로 즉시 스킵할 수 있다.

## 목표 플레이 경험

플레이어는 손님의 대사를 보고 다음처럼 판단한다.

- "다쳤다", "상처", "회복" 같은 말이 나오면 회복 포션
- "잠", "불면", "조용히 재우고 싶다" 같은 말이 나오면 수면 포션
- "시험", "고백", "승부", "운" 같은 말이 나오면 행운 포션
- "무거운 일", "힘", "광산", "갑옷" 같은 말이 나오면 힘 포션

정확한 키워드를 외우는 게임이라기보다, 손님의 상황을 읽고 자연스럽게 포션을 고르는 게임이 되어야 한다.

## 화면 구성

### 기본 제작 화면

한 화면 안에서 주문, 제작, 제출이 모두 이루어진다.

- 중앙: 큰 가마솥
- 하단 또는 좌측: 재료 카운터
- 우측: 완성 포션 제출대
- 상단: 날짜, 골드, 현재 손님 번호
- 전면 또는 우측: 손님 캐릭터와 말풍선

### 권장 레이아웃

```text
+--------------------------------------------------+
| Day 1                         Gold 20   2 / 5    |
|                                                  |
|             [손님 캐릭터]                         |
|        "요즘 잠을 통 못 자겠어요..."              |
|                                                  |
|     [재료 카운터]        [가마솥]      [제출대]   |
|  버섯 이슬 꽃잎 뿌리 별가루    부글부글      포션 |
|                                                  |
|              [젓기/제작] [초기화] [제출]          |
+--------------------------------------------------+
```

## 기본 루프

1. 손님이 등장한다.
2. 손님이 말풍선으로 주문을 말한다.
3. 플레이어가 재료 2개를 선택해 가마솥에 넣는다.
4. 제작 버튼을 누르면 조합 결과가 나온다.
5. 완성 포션을 제출한다.
6. 정답이면 높은 보상과 긍정 반응을 받는다.
7. 오답이면 낮은 보상과 실망 반응을 받는다.
8. 다음 손님이 등장한다.
9. 하루 손님을 모두 처리하면 정산 화면으로 넘어간다.

## 조작 방식

MVP 기준으로 가장 단순한 방식을 사용한다.

- 재료 드래그: 재료 아이콘을 가마솥으로 끌어다 놓으면 추가
- 재료는 최대 3개까지 넣을 수 있음 (최소 2개)
- 같은 재료를 두 번 넣는 것은 허용하지 않음
- 넣은 재료를 다시 누르면 취소(제거) — 별도의 초기화 버튼은 없음
- 제작 버튼: 선택한 재료 2~3개로 포션 생성
- 제출 버튼: 완성된 포션을 현재 손님에게 전달

드래그 앤 드롭은 재미있지만 구현량이 늘어나므로 1차 버전에서는 클릭 방식이 우선이다.

| strength | 힘 포션 | 불타는도마도 + 산삼 |
| sleep | 수면 포션 | 번데기 + 도토링 |

조합 순서는 상관없다. 현재 손님이 원하는 포션의 필요 재료는 우측 "손님 주문" 패널 하단에 아이콘으로 표시된다.

예시:

- 산삼 + 포도 = 회복 포션
- 포도 + 산삼 = 회복 포션

정해진 조합이 아니면 `실패한 포션`이 나온다.

## 실패한 포션

잘못된 재료 조합으로 만든 포션.

- 이름: 수상한 포션
- 색상: 회색 또는 탁한 보라색
- 제출 가능
- 어떤 주문에도 정답 처리되지 않음
- 오답 보상만 지급

## 하루 진행

처음 3일은 튜토리얼처럼 포션 종류를 서서히 늘린다.

| 날짜 | 손님 수 | 등장 포션 | 목표 골드 |
| --- | --- | --- | --- |
| Day 1 | 3명 | 회복, 힘 | 20 |
| Day 2 | 4명 | 회복, 힘, 수면 | 30 |
| Day 3 | 5명 | 회복, 힘, 수면, 행운 | 40 |
| Day 4+ | 5명 | 전체 랜덤 | 40 이상 |

MVP에서는 Day 4 이후 무한 반복 또는 Day 5까지만 제공해도 충분하다.

## 보상 규칙

- 정답 제출: 10골드
- 오답 제출: 2골드
- 실패한 포션 제출: 2골드
- 하루 목표 골드 달성: 다음 날 진행
- 하루 목표 골드 미달: 같은 날 다시 도전

호감도, 팁, 콤보 보상은 MVP 이후 기능으로 남긴다.

## 상태값 예시

```js
const gameState = {
  day: 1,
  gold: 0,
  currentCustomerIndex: 0,
  customersToday: [],
  selectedIngredients: [],
  craftedPotion: null,
  phase: "order"
};
```

권장 phase:

- `order`: 손님 주문을 보여주는 상태
- `brewing`: 재료를 선택하는 상태
- `crafted`: 포션이 완성된 상태
- `result`: 제출 결과를 보여주는 상태
- `day_end`: 하루 정산 상태

## 조합 판정 규칙

```js
function getPotionByIngredients(selectedIngredients) {
  const sortedSelected = [...selectedIngredients].sort();

  return recipes.find((recipe) => {
    const sortedRecipe = [...recipe.ingredients].sort();
    return sortedRecipe[0] === sortedSelected[0]
      && sortedRecipe[1] === sortedSelected[1];
  }) || {
    potionId: "failed",
    name: "수상한 포션",
    ingredients: selectedIngredients,
    color: "#7b7280"
  };
}
```

- 배경: 따뜻한 나무색, 어두운 초록, 약한 금색 조명
- 가마솥: 짙은 회색/검정
- 회복 포션: 빨강
- 수면 포션: 파랑/남색
- 행운 포션: 노랑/금색
- 힘 포션: 주황/빨강

## 화면별 요구사항

### 메인 제작 화면

표시 요소:

- 날짜
- 골드
- 현재 손님 진행도
- 손님 이름
- 손님 대사
- 재료 목록
- 가마솥
- 선택한 재료
- 제작된 포션
- 초기화/제작/제출 버튼

행동:

- 주문 대사 확인
- 재료 선택
- 재료 2개가 선택되면 제작 가능
- 제작 후 제출 가능

### 결과 화면 또는 결과 패널

표시 요소:

- 정답/오답 여부
- 손님 반응
- 획득 골드
- 다음 손님 버튼

행동:

- 다음 손님으로 이동
- 마지막 손님이면 하루 정산으로 이동

### 하루 정산 화면

표시 요소:

- Day N 종료
- 오늘 번 골드
- 목표 골드
- 성공/실패 여부

행동:

- 성공 시 다음 날 시작
- 실패 시 같은 날 재도전

## 튜토리얼 방식

별도의 긴 튜토리얼은 만들지 않는다.

Day 1 첫 손님만 매우 직접적으로 말한다.

예시:

- "다쳐서 회복할 약이 필요해요. 생기 있어 보이는 재료가 좋겠죠?"

Day 1에는 주문 난도를 낮추고, Day 2부터 조금 더 자연스러운 대사를 사용한다.

## 난이도 설계

난이도는 대사 표현만으로 조절한다.

쉬운 주문:

- "상처가 빨리 낫는 약 주세요."
- "잠이 오는 포션이 필요해요."

보통 주문:

- "어제 던전에서 구른 뒤로 몸이 욱신거려요."
- "침대에 누워도 정신이 너무 또렷해요."

조금 애매한 주문:

- "내일 중요한 일이 있어서 컨디션을 끌어올리고 싶어요."

MVP에서는 너무 애매한 주문은 피한다.

## 구현 우선순위

1. 재료, 레시피, 주문 데이터 작성
2. 제작 화면 레이아웃
3. 재료 선택과 가마솥 상태 구현
4. 레시피 판정 구현
5. 손님 주문과 정답 포션 연결
6. 제출 및 보상 처리
7. 하루 진행 처리
8. 최소한의 스타일 적용
9. 간단한 애니메이션과 효과음은 여유가 있을 때 추가

## 완료 기준

다음 조건을 만족하면 1차 완성으로 본다.

- 플레이어가 Day 1부터 Day 3까지 진행할 수 있다.
- 5개 재료 중 2개를 선택해 포션을 만들 수 있다.
- 4종 포션이 모두 제작 가능하다.
- 손님의 주문과 포션 제출 결과가 정상 판정된다.
- 정답과 오답에 따라 다른 반응과 골드가 나온다.
- 하루 손님 수가 끝나면 정산 화면이 나온다.
- 목표 골드에 따라 다음 날 또는 재도전으로 분기된다.

## 이후 확장 아이디어

MVP가 안정적으로 완성된 뒤에만 고려한다.

- 재료 1개 추가
- 포션 2종 추가
- 손님별 단골 대사
- 자객이 가끔 손님인척 오는 이벤트로 독 버섯으로 포션을 만들어서 몰래 제출하는 이벤트
- 가게 꾸미기
- 간단한 업그레이드
- 빠른 제작 보너스
- 손님 만족도
- 랜덤 특수 주문
- 스토리 이벤트

## 개발 메모

이 프로젝트는 기획보다 완성이 중요하다. 처음부터 시스템을 크게 만들지 말고, 플레이 가능한 작은 루프를 먼저 완성해야 한다.

가장 중요한 재미는 손님의 말을 읽고 "이 포션이겠네"라고 판단한 뒤, 알맞은 재료를 넣어 결과를 확인하는 순간이다. 모든 구현은 이 재미를 방해하지 않는 방향으로 단순하게 유지한다.
